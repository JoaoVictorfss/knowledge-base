Indentação : Indentar o código facilita a compreensão. Eslint e Prettier como exemplos de ferramentas para automatizar. Variáveis : As variáveis ​​devem ser designadas de maneira intuitiva, com nomes pequenos e de fácil entendimento. Quando uma variável tiver uma função de representar uma coleção de objetos, tais como array, lista ou conjunto, deixe o seu nome no plural (ex: nomes, valores). Para nomes de variáveis com palavras compostas, apenas a primeira letra da segunda palavra deve estar em maiúsculo. Exemplos: nome, urlExistente e usuarios. Constantes: O nome das constantes sempre deve ser declarado em letras maiúsculas, separadas por underscores “_” no caso de existirem múltiplas palavras. Exemplo: public static final String PROFESSOR = "role_professor"; Funções: Nomear funções de maneira intuitiva, não se importe com o tamanho do nome.
